## 2026-05-30 - [Performance] NeuralNetworkDiagram re-renders & expensive calculations
**Learning:** In SVG-based network diagram components, recalculating node positions and mapping connectivity arrays inside the render function on every render creates massive array allocations and object instantiations, leading to unnecessary garbage collection pauses. Also, doing DOM manipulation inside `useEffect` combined with lack of memoization makes the component very heavy.
**Action:** Extract layout calculation loops and nested array map operations into independent `useMemo` blocks. Wrap the `getNodePosition` helper function in `useCallback` (or inline it in the memo block) to provide stable dependencies and prevent triggering layout recalculations during re-renders unless the configuration changes.
