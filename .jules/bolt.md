## 2024-06-12 - [Array.from().flat() Float32Array Instantiation Bottleneck]
**Learning:** Instantiating `Float32Array` by mapping over an array and then calling `.flat()` (e.g., `new Float32Array(Array.from({ length: n }, ...).flat())`) inside React render functions causes severe memory allocation overhead and is >10x slower than pre-allocating the `Float32Array` size and using a direct `for` loop to populate elements. This is especially problematic in 3D scenes where buffers might be rebuilt unnecessarily.
**Action:** Always pre-allocate the `Float32Array` length (e.g., `new Float32Array(count * 3)`) and use direct indexing to populate it. Furthermore, memoize these buffer calculations with `useMemo` with empty dependency arrays if they represent static positions to prevent re-evaluation on re-renders.
