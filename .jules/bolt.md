## 2025-03-08 - Declarative SVG vs Imperative DOM manipulation in React
**Learning:** Found a component (`NeuralNetworkDiagram`) that used a `useEffect` and `querySelectorAll` to imperatively calculate distances between dynamically generated SVG `<line>` coordinates and assign inline styles and animation attributes. This forces a double render/repaint by manually modifying the DOM after React has committed the tree.
**Action:** When using CSS animations on SVG elements that depend on dynamic visual properties (like `stroke-dashoffset` needing the exact line length), pre-calculate these values during the render phase and use `useMemo`. Apply them declaratively via inline styles, using CSS variables to sync the calculated property with the `@keyframes` definition, avoiding post-render imperative DOM queries.
