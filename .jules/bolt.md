## 2025-06-07 - Declarative SVG Animation Optimization in React
**Learning:** Imperatively querying SVG elements (`querySelectorAll`) and mutating their styles within a `useEffect` is a major performance anti-pattern in React. It breaks referential equality and forces synchronous layout calculations (`Math.sqrt(...)` for line length) during the render cycle, which causes UI thrashing, especially for animated connections. Setting both `strokeDasharray` and `strokeDashoffset` to the line's length hides it if `animated={false}` because the offset never animates to 0.
**Action:** Extract node count defaults outside the component signature to prevent breaking `useMemo`. Pre-calculate layout properties (like `x, y`, and line `length`) within `useMemo` hooks, and bind them declaratively via React's `style` prop. For CSS keyframes dependent on dynamic JS calculations, use custom CSS variables (`--path-length`) to bridge the gap seamlessly, and conditionally apply dash properties only when `animated` is true.
